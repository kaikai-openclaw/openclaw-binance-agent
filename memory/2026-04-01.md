# 2026-04-01

## 性能分析报告（用户提出，仅记录不改）

### 问题 1: Python 版本混乱
- 系统 `/usr/bin/python3` = 3.9.6，不支持 `dict | None`（PEP 604）
- homebrew `python3` = 3.14.3，`.venv/bin/python3` = 3.14.3
- `binance_fapi.py` 大量使用 `dict | None`、`str | None`、`list[dict]` 等 3.10+ 语法
- 当前 PATH 优先 homebrew，所以 `python3` 能跑。但如果 OpenClaw 用 `/usr/bin/python3` 就会 SyntaxError
- 影响文件: `binance_fapi.py`, `trading_agents_adapter.py`, `base.py`, `skill1_collect.py`, `paper_backtest.py` 等

### 问题 2: exec 冷启动架构
- SKILL.md 所有命令都是 `python3 -c "..."` 模式，每次新进程
- 每次调用开销: 解释器启动 ~200-500ms + 依赖加载 ~100-300ms + SSL ~50ms
- 没有模块缓存、连接池复用、SSL session 复用
- `BinanceFapiClient.__init__` 每次创建新 `requests.Session`
- `_load_symbol_info()` 每次冷启动都要重新请求 `exchangeInfo`

### 问题 3: 重试 + 限流开销
- `calculate_backoff` 序列 [1, 2, 4, 8, 16]，最多 5 次
- 最坏情况纯等待 31 秒
- 加上网络抖动，单次查询可能 20+ 秒

### 潜在优化方向（备忘，未实施）
1. 版本兼容: 把 `dict | None` 改回 `Optional[dict]`（兼容 3.9），或统一强制 3.14
2. 长驻进程: 封装成 gRPC/HTTP daemon，首次加载后调用 ms 级
3. 减少重试: 降低 MAX_RETRIES 或改异步并发
4. 模块预加载: 用 exec background 模式保持进程复用
5. exchangeInfo 缓存: 持久化到 SQLite，避免每次冷启动请求
