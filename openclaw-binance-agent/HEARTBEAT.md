# 心跳与定时任务

心跳不是简单回复在线状态，而是执行低风险巡检。任何会触发实盘交易的任务必须走专用脚本和硬编码风控。

## 高频巡检

### 1. 账户与保护单检查

- 建议频率：每 5-15 分钟。
- 推荐脚本：`.venv/bin/python3 skills/binance-trading/scripts/check_account.py`
- 必看字段：总资金、可用保证金、未实现盈亏、持仓保证金占比、杠杆、普通挂单、Algo 止盈止损条件单。
- 如果发现无持仓残留保护单、重复保护单、缺少止损或止盈，要在报告中明确标记。

### 2. 日亏损与 Paper Mode

- 建议频率：每 30 分钟。
- 日亏损达到 5% 时，系统必须停止实盘并持久化 Paper Mode。
- Paper Mode 状态来自 `RiskController` 的 SQLite runtime state，不要直接改私有字段。

## 超跌交易定时任务

固定入口：

```bash
.venv/bin/python3 skills/binance-trading/scripts/run_oversold_cron.py --fast --format markdown
```

输出必须保持结构一致，包含：

- 扫描结果：全市场数量、基础过滤、超跌候选、最终输出。
- 评级结果：所有已分析币种、评分、方向、置信度、是否达标。
- 交易结果：本轮开仓、风控拒绝、执行失败、服务端已平仓同步数量。
- 持仓明细：方向、数量、入场价、当前价、涨跌幅、浮盈亏、名义价值、保证金、资金占比、杠杆、保证金收益率。
- 保护单状态：止损/止盈数量、重复保护单、残留条件单。
- 账户与风险：总资金、可用保证金、持仓资金占比、日亏损、风控阈值。

定时任务不要让模型自由改写脚本输出；脚本生成的 Markdown 就是最终报告。

## 主动通知条件

- 发生真实开仓、平仓或风控拒绝。
- Binance 服务端 STOP/TAKE_PROFIT 触发并同步到 MemoryStore。
- 日亏损接近或达到 5%。
- 任一持仓缺少止损或止盈保护。
- 发现重复保护单或无持仓残留条件单。
- Binance API 返回精度、余额、签名、限流或 IP 封禁错误。

## 安静条件

- 无持仓、无交易、无异常时，只写日志或简短状态。
- 深夜 23:00-08:00 除非实盘风险事件，不主动打扰。
